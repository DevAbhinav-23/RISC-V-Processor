# RISC-V-Processor
We will be building a processor

## Arithmetic Logic Unit (ALU)

The **ALU** is the primary execution block of the processor. It performs arithmetic and logical operations on two 64-bit input operands based on the control signals generated by the ALU Control block. It takes data from the registers, processes it according to the instruction opcode, and outputs the result to be stored or used for memory addressing.

---

### Inputs
| Signal | Width | Source | Description |
| :--- | :--- | :--- | :--- |
| **Operand A** | 64 bits | Register File | First source operand. |
| **Operand B** | 64 bits | Register File / Immediate | Second source operand. |
| **ALU Control** | 4 bits | ALU Control Unit | Determines the specific operation to perform. |

### Outputs
| Signal | Width | Destination | Description |
| :--- | :--- | :--- | :--- |
| **ALU Result** | 64 bits | Data Bus / Memory | The numerical or logical output of the operation. |
| **Zero Flag** | 1 bit | Control Unit | Asserted high (`1`) if the `ALU Result` is exactly zero. |

---

### Logic Behavior
The `zero_flag` is calculated using a NOR-reduction of the result bus.
This flag is essential for implementing instructions such as `BEQ` (Branch if Equal), where the equality of two numbers is determined by subtracting them and checking if the result is zero.
